# 第一章 組譯器基礎概念

## 1.1 組譯器的角色與定義

組譯器（Assembler）是將組譯語言（Assembly Language）翻譯成機器碼（Machine Code）的軟體工具。它是編譯器鏈中最接近硬體的環節，扮演著至關重要的角色。

### 組譯器的核心功能

1. **符號解析**：將標籤（Label）轉換為實際記憶體位址
2. **指令翻譯**：將助記符號（Mnemonic）轉換為二進位機器碼
3. **位址計算**：處理相對位址、絕對位址與間接位址
4. **資料轉換**：處理常數、字串與資料定義

## 1.2 組譯器與編譯器的比較

| 特性 | 組譯器 | 編譯器 |
|-----|-------|-------|
| 輸入語言 | 組譯語言 | 高階語言 |
| 輸出語言 | 機器碼 | 目標碼/組譯碼 |
| 翻譯比率 | 1:1 | 多:1 |
| 最佳化程度 | 較少 | 較多 |
| 硬體相依性 | 高 | 中等 |

## 1.3 組譯器的發展歷史

### 早期發展（1950年代）

- 1952年：IBM 推出第一個自動組譯器
- 1955年：Symbolic Assembler 問世

### 中期演進（1960-1980年代）

- 巨集組譯器（Macro Assembler）出現
- 條件組譯功能加入
- 交叉組譯器開始普及

### 現代組譯器（1990年代至今）

- 支援多種CPU架構
- 整合開發環境
- 與高階語言緊密結合

## 1.4 組譯器的應用領域

1. **系統程式設計**：作業系統、驅動程式
2. **嵌入式系統**：資源受限的環境
3. **效能關鍵程式**：需要精確控制的應用
4. **逆向工程**：軟體分析與破解
5. **教學用途**：了解計算機運作原理

## 1.5 組譯器的分類

### 單遍與多遍組譯器

- **單遍組譯器**：一次性掃描完成，無法處理向前引用
- **多遍組譯器**：多次掃描，可處理複雜的符號依賴

### 交叉與本地組譯器

- **交叉組譯器**：在主機上產生目標機器的程式碼
- **本地組譯器**：在目標機器上執行

### 基本與高階組譯器

- **基本組譯器**：僅支援基本指令與指示
- **高階組譯器**：支援巨集、條件組譯等進階功能
