# 第二章 組譯語言基礎

## 2.1 指令集架構介紹

指令集架構（Instruction Set Architecture, ISA）是處理器設計的核心規範，定義了處理器支援的指令集合及其行為。常見的指令集架構包括：

- **x86**：複雜指令集計算（CISC），支援豐富的指令與定址模式
- **ARM**：精簡指令集計算（RISC），強調效率與功耗
- **MIPS**：經典RISC架構，常用於教學環境
- **RISC-V**：開放式ISA，設計簡潔且可擴展

## 2.2 指令格式與定址模式

### 指令格式

指令通常由以下欄位組成：
- **運碼（Opcode）**：指定要執行的操作
- **運算元（Operand）**：指定操作涉及的資料

### 定址模式

常見的定址模式包括：

1. **立即定址**：運算元是常數值
   ```
   MOV AX, 5      ; 將5載入AX
   ```

2. **暫存器定址**：運算元是暫存器
   ```
   MOV AX, BX     ; 將BX內容複製到AX
   ```

3. **直接定址**：運算元是記憶體位址
   ```
   MOV AX, [1000h]; 將位址1000h的內容載入AX
   ```

4. **間接定址**：運算元是存有記憶體位址的暫存器
   ```
   MOV AX, [BX]   ; 將BX所指向的記憶體內容載入AX
   ```

5. **基底加偏移定址**：使用基底暫存器加上常數偏移
   ```
   MOV AX, [BX+4] ; 將BX+4位址的內容載入AX
   ```

## 2.3 資料處理指令

資料處理指令用於執行算術、邏輯與資料傳輸操作：

### 算術指令
- **ADD**：加法
- **SUB**：減法
- **MUL/DIV**：乘除法
- **INC/DEC**：遞增/遞減

### 邏輯指令
- **AND**：位元AND
- **OR**：位元OR
- **XOR**：位元XOR
- **NOT**：位元NOT

### 資料傳輸
- **MOV**：資料複製
- **PUSH/POP**：堆疊操作
- **LEA**：載入有效位址

## 2.4 控制轉移指令

控制轉移指令改變程式的執行流程：

### 跳躍指令
- **JMP**：無條件跳躍
- **JE/JNE**：條件跳躍（相等/不相等）
- **JG/JL**：條件跳躍（大於/小於）

### 呼叫與返回
- **CALL**：呼叫副程式
- **RET**：返回呼叫點
- **INT**：軟體中斷

## 2.5 記憶體存取指令

記憶體存取指令負責在暫存器與記憶體之間傳輸資料：

```
; 載入記憶體內容到暫存器
LDR R0, [R1]         ; 將R1指向的記憶體內容載入R0

; 儲存暫存器內容到記憶體
STR R0, [R1]         ; 將R0的內容儲存到R1指向的記憶體

; 多重載入/儲存
LDMIA R0!, {R1-R3}   ; 連續載入多個暫存器
STMIA R0!, {R1-R3}   ; 連續儲存多個暫存器
```

## 2.6 巨集與條件組譯

### 巨集（Macro）

巨集是一種程式碼重用機制，可定義可重複使用的指令序列：

```assembly
MACRO PRINT_STRING str
    LEA DX, str
    MOV AH, 09h
    INT 21h
ENDMACRO

; 使用巨集
PRINT_STRING msg1
PRINT_STRING msg2
```

### 條件組譯

條件組譯允許根據條件包含或排除程式碼：

```assembly
IFdef DEBUG
    MOV CX, 0        ; 偵錯模式專用碼
ELSE
    MOV CX, 1        ; 正式版本
ENDIF
```
