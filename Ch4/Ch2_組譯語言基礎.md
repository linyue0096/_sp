# 第二章 組譯語言基礎

## 2.1 組譯語言介紹

組譯語言是一種低階程式語言，它使用助記符號（Mnemonic）來表示機器指令，讓程式設計師能夠以較人類可讀的方式撰寫程式。

### 組譯語言的特性

1. **直接對應硬體**：每條指令通常對應一條機器指令
2. **硬體相依性**：依賴特定CPU架構與指令集
3. **低階控制**：可直接控制暫存器、記憶體與周邊設備
4. **效率高**：產生的程式碼精簡且執行快速

### 組譯語言的基本結構

```
; 註解
LABEL:  INSTRUCTION  OPERAND  ; 註解
```

## 2.2 指令集架構基礎

### 常見指令集架構

1. **x86/x64**：Intel與AMD處理器
2. **ARM**：行動裝置與嵌入式系統
3. **MIPS**：網路設備與學術用途
4. **RISC-V**：開放來源RISC架構

### 指令格式

```
[標籤:]  操作碼  [運算元1, 運算元2, 運算元3]  [;註解]
```

### 常見指令類型

- **資料傳送指令**：MOV, LOAD, STORE
- **算術運算指令**：ADD, SUB, MUL, DIV
- **邏輯運算指令**：AND, OR, XOR, NOT
- **控制轉移指令**：JMP, CALL, RET, BRANCH
- **位元操作指令**：SHIFT, ROTATE, TEST

## 2.3 標籤與符號解析

### 標籤的用途

1. **標記位址**：作為跳轉目標
2. **定義常數**：資料位置參考
3. **建立程序**：函數入口點

### 標籤規則

- 大小寫敏感（或依組譯器而定）
- 不能使用保留字
- 通常以字母或底線開頭

### 符號表結構

```python
SymbolTable = {
    "label_name": {
        "address": 0x1000,
        "type": "code" | "data",
        "scope": "global" | "local"
    }
}
```

## 2.4 基本的組譯指令

### x86 基本範例

```asm
; 資料傳送
MOV AX, 5          ; 將5載入AX
MOV BX, AX          ; 將AX複製到BX

; 算術運算
ADD AX, BX          ; AX = AX + BX
SUB CX, 10          ; CX = CX - 10
MUL DX              ; AX = AX * DX

; 邏輯運算
AND AX, 0xFF        ; 遮罩運算
OR BX, 0x01         ; 設定位元

; 控制轉移
JMP label           ; 無條件跳轉
JE equal_label      ; 等於時跳轉
CALL procedure      ; 呼叫程序
RET                 ; 返回
```

## 2.5 資料類型與表示法

### 整數表示

- **十進位**：100, -50
- **十六進位**：0x64, 0xFF
- **二進位**：1100100b
- **八進位**：0144

### 字元與字串

```asm
DB 'A'              ; 位元組資料
DW "AB"             ; 字串資料
DB "Hello, World!"  ; 字串常數
```

### 資料定義指示

```asm
DB  1               ; 定義位元組 (8位元)
DW  1               ; 定義字 (16位元)
DD  1               ; 定義雙字 (32位元)
DQ  1               ; 定義四字 (64位元)
```

## 2.6 組譯指示（Directive）

### 區段定義

```asm
section .data       ; 資料區段
section .text       ; 程式碼區段
section .bss        ; 未初始化資料區段
```

### 記憶體對齊

```asm
ALIGN 4             ; 4位元組對齊
```

### 包含檔案

```asm
%include "macro.inc"  ; 包含巨集檔案
```
