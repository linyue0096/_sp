# 附錄A：專有名詞解釋

## 1. 基礎術語

### 組譯器（Assembler）
組譯器是將組譯語言翻譯成機器碼的程式。它讀取包含助記符和符號的原始程式碼，輸出可由處理器直接執行的二進制指令。組譯器完成了從人類可讀的符號表達到機器可執行格式的關鍵轉換。

### 組譯語言（Assembly Language）
組譯語言是一種低階程式語言，使用助記符來表示機器指令。它是機器碼的符號化表示，每條組譯指令通常對應一條機器指令。組譯語言提供了對硬體的直接控制能力，但需要程式設計師對目標架構有深入了解。

### 機器碼（Machine Code）
機器碼是處理器可以直接執行的二進制指令格式。每種處理器架構都有其特定的機器碼指令集，機器碼是組譯器的最終輸出目標。

### 助記符（Mnemonic）
助記符是用於表示機器指令的縮寫名稱。例如，「MOV」表示「move」指令，「ADD」表示加法指令。助記符使得程式設計師無需記憶複雜的機器碼數字。

### 目標檔案（Object File）
目標檔案是組譯器的輸出檔案，包含已編譯的機器碼以及符號表、重定位資訊等元資料。目標檔案需要經過連結器處理才能形成可執行檔。

## 2. 指令與架構術語

### 指令集架構（Instruction Set Architecture，ISA）
指令集架構定義了處理器支援的所有指令、暫存器、記憶體模型和處理器狀態。常见的ISA包括x86、ARM、MIPS和RISC-V。ISA是硬體與軟體之間的抽象層介面。

### CISC（Complex Instruction Set Computer）
CISC是一種指令集設計理念，特點是指令數量多、指令格式複雜、指令長度可变。CISC架構的目標是通過複雜的單一指令完成多種操作，減少程式碼量。x86是典型的CISC架構。

### RISC（Reduced Instruction Set Computer）
RISC是一種指令集設計理念，特點是指令數量少、指令格式固定、指令長度统一。RISC架構強調指令執行效率，通常支援管線化執行。ARM和MIPS是典型的RISC架構。

### 暫存器（Register）
暫存器是處理器內部的高速記憶體單元，用於臨時存儲資料和指令運算元。暫存器的訪問速度遠快於主記憶體，因此在效能關鍵的程式碼中廣泛使用。不同架構有不同數量和類型的暫存器。

### 定址模式（Addressing Mode）
定址模式定義了如何指定指令運算元的位址。常見的定址模式包括立即定址、暫存器定址、直接定址、間接定址和基底加偏移定址等。

## 3. 程式設計術語

### 標籤（Label）
標籤是用於標識程式位置（如程式碼行或資料位置）的符號名稱。標籤允許程式設計師使用符號名稱來引用位置，而無需知道具體的記憶體位址。組譯器會將標籤轉換為具體的位址值。

### 偽指令（Directive/Pseudo-instruction）
偽指令是給組譯器的特殊指示，不會產生機器碼。偽指令用於控制組譯過程，如定義資料、設定區段、控制條件組譯等。常見的偽指令包括DB、DW、EQU、ORG等。

### 巨集（Macro）
巨集是一種程式碼重用機制，允許定義可重複使用的指令序列。使用巨集可以減少重複程式碼，提高程式碼的可讀性和可維護性。組譯器在處理巨集時會進行巨集展開。

### 條件組譯（Conditional Assembly）
條件組譯允許根據預先定義的條件選擇性地包含或排除程式碼。這對於開發不同平台版本、建立偵錯版本和進行跨平台移植非常有用。

## 4. 編譯器理論術語

### 詞彙分析（Lexical Analysis）
詞彙分析是編譯過程的第一階段，將原始程式碼字元流分割為詞彙單元（Token）。詞彙分析器識別指令助記符、標籤、運算元、數值常量等語言要素。

### 語法分析（Syntax Analysis）
語法分析驗證詞彙單元序列是否符合語法規則，並建立語法樹或抽象語法樹來表示程式的結構。語法分析確保程式碼遵循語言語法規範。

### 符號表（Symbol Table）
符號表是存儲程式碼中所有符號資訊的資料結構，包括符號名稱、類型、位址和作用域等資訊。符號表貫穿整個編譯過程，用於解析符號引用。

### 位址解析（Address Resolution）
位址解析是將符號引用轉換為具體記憶體位址的過程。組譯器需要計算標籤的位址、處理相對位址和絕對位址，以及解析跨模組的符號引用。

### 重定位（Relocation）
重定位是調整目標檔案中的位址以適應最終載入位置的過程。由於目標檔案在連結前不知道具體的載入位址，需要在連結時進行重定位調整。

## 5. 連結與載入術語

### 連結器（Linker）
連結器將多個目標檔案和庫檔案合併為單一的可執行檔案。連結器解析跨檔案的符號引用，進行符號重定位，並處理區段的合併和配置。

### 載入器（Loader）
載入器負責將可執行檔案載入記憶體並準備執行。載入器會處理區段的記憶體配置、相對位址的調整，並將控制權轉交給程式的入口點。

### 符號解析（Symbol Resolution）
符號解析是查找和確認符號定義的過程。當程式碼引用其他模組中定義的符號時，連結器需要找到該符號的定義位置並建立正確的引用。

### 外部符號（External Symbol）
外部符號是在其他模組中定義、可被當前模組引用的符號。EXTERN和PUBLIC（或GLOBAL）偽指令用於宣告和導出外部符號。

## 6. 進階術語

### 交叉組譯（Cross Assembly）
交叉組譯是指在一种主機平台上生成另一種目標平台可執行碼的過程。這在嵌入式系統開發中特別重要，因為目標設備可能無法運行本地組譯器。

### JIT編譯（Just-In-Time Compilation）
JIT編譯是一種在程式執行時動態將中間碼（如bytecode）翻譯為機器碼的技術。JIT編譯結合了解譯的靈活性和編譯的效能，常用於Java和.NET執行環境。

### 反組譯（Disassembly）
反組譯是將機器碼轉換回組譯語言的過程。反組譯器分析二進制指令並產生人類可讀的組譯程式碼，常用於軟體分析、偵錯和安全研究。

### 指令排程（Instruction Scheduling）
指令排程是最佳化指令執行順序以提高管線效率的技術。通過重新排列指令順序，可以減少管線停滯和資料相依性問題。

### SIMD（Single Instruction Multiple Data）
SIMD是一種平行處理技術，允許單條指令同時處理多筆資料。現代處理器透過SIMD指令（如x86的SSE/AVX、ARM的NEON）加速向量運算。

## 7. 檔案格式術語

### ELF（Executable and Linkable Format）
ELF是Unix和Linux系統標準的可執行檔案和目標檔案格式。ELF格式支援多種區段類型，包括程式碼、資料、符號表和重定位資訊。

### COFF（Common Object File Format）
COFF是Windows系統早期使用的目標檔案格式。COFF定義了目標檔案的結構，包括區段表、符號表和行號資訊。

### PE（Portable Executable）
PE是Windows可執行檔案的格式，基於COFF並進行了擴展。PE格式用於EXE、DLL和SYS等檔案類型。

### DWARF
DWARF是一種調試資訊格式，用於在目標檔案中存儲源程式級的調試資訊。DWARF包含原始碼行號、變數位置和類型資訊。
