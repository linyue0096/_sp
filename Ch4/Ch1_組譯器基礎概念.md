# 第一章 組譯器基礎概念

## 1.1 組譯器的定義與角色

組譯器（Assembler）是將組譯語言（Assembly Language）翻譯成機器碼的程式工具。它扮演著硬體與軟體之間的橋樑角色，使程式設計師能夠以人類可讀的符號式指令來撰寫程式，而無需直接處理繁瑣的二进制機器碼。

組譯器的核心功能包括：
- 將助記符（Mnemonic）轉換為對應的機器指令
- 解析符號式位址並進行位址計算
- 管理程式碼與資料的配置
- 處理巨集與條件式組譯

## 1.2 組譯器與編譯器的比較

組譯器與編譯器雖然同屬翻譯工具，但在設計理念與實現方式上存在顯著差異：

| 特性 | 組譯器 | 編譯器 |
|------|--------|--------|
| 翻譯單位 | 一對一對應 | 多對一對應 |
| 輸出 | 機器碼 | 中間碼或機器碼 |
| 最佳化 | 有限 | 高度 |
| 移植性 | 低 | 高 |
| 硬體相依性 | 高 | 低 |

## 1.3 組譯語言的特性

組譯語言具有以下特性：

1. **低階性**：直接操作硬體資源，如暫存器、記憶體與周邊裝置
2. **硬體相依性**：不同處理器架構有不同的指令集
3. **可預測性**：程式行為可精確控制，無隱藏成本
4. **效率性**：可直接產生最佳化的機器碼

## 1.4 組譯器的發展歷史

組譯器的發展經歷了以下階段：

- **1950年代**：第一代組譯器，僅支援基本指令助記符
- **1960年代**：引入巨集功能，增強程式碼重用
- **1970年代**：條件式組譯與交叉組譯出現
- **1980年代**：最佳化技術與符號偵錯功能
- **1990年代至今**：JIT編譯與反組譯技術的發展

## 1.5 組譯器的工作流程

組譯器的工作流程可分為以下階段：

```
原始碼 → 詞彙分析 → 語法分析 → 語意分析 → 產生目標碼 → 輸出
```

1. **詞彙分析**：識別指令助記符、運算元、標籤與偽指令
2. **語法分析**：驗證指令格式的正確性
3. **語意分析**：進行位址計算與型別檢查
4. **目標碼產生**：生成二進制機器碼
