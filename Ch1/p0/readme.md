* p0 編譯器技術文件：while 迴圈實作與函數呼叫機制
本文件分為兩個主要部分：首先說明如何在現有的 p0 編譯器架構中加入 while 迴圈語法；接著，深入探討為了支援自定義函數（func），底層堆疊機（Stack Machine）的函數呼叫機制是如何運作的。
* 第一部分：為 p0 編譯器加上 while 語法
在編譯器中實作 while 迴圈，必須穿越前端的三個核心階段：詞法分析、語法分析，以及程式碼生成。
* 1. 詞法分析器 (Lexer)
Lexer 的任務是將字元流轉換為 Token。我們需要讓掃描器認識 while 這個關鍵字，並將其標記為 TK_WHILE。
***
if (strncmp(src, "while", 5) == 0 && !isalnum(src[5])) {
    current_token.type = TK_WHILE;
    src += 5;
}
***
* 2. 語法分析器 (Parser)
Parser 負責將 Token 組織成抽象語法樹 (AST)。while 迴圈的語法規則通常是 while ( 條件式 ) { 主體 }。因此，我們會建立一個 AST_WHILE 節點，它包含兩個重要的子分支：
- cond (條件節點)：儲存括號內的條件運算（如 i < 10）。
- body (主體節點)：儲存大括號內的多行程式碼（通常是一個 AST_BLOCK 節點，裡面串接了多個陳述句）。
* 3. 程式碼生成器 (Code Generator)
這是最核心的步驟。堆疊機本身沒有 while 這種高階結構，我們必須利用「標籤 (Label)」與「跳躍指令 (Jump)」來控制執行流程：
1. 生成起始標籤：標示迴圈的開頭（例如 L0:）。
2. 生成條件運算：編譯 cond 節點，將判斷結果推入堆疊。
3. 條件跳躍：加入 JUMP_IF_FALSE L1 指令。如果條件為假，直接跳到迴圈結束的標籤 L1。
4. 生成迴圈主體：編譯 body 節點的程式碼。
5. 無條件跳躍：在主體結尾加入 JUMP L0，強制跳回步驟 1 重新判斷。
6. 生成結束標籤：標示迴圈的終點（例如 L1:）。
* 第二部分：函數呼叫機制 (Function Call) 是怎麼運作的？
若要讓編譯器支援 func add(a, b) 這樣的語法，就必須理解虛擬機的堆疊影格 (Stack Frame / Activation Record) 機制。
當程式呼叫一個函數時，不能只是跳轉到該函數的記憶體位址，還必須處理「參數傳遞」、「區域變數分配」以及「如何準確返回原處」。為此，每次呼叫函數，系統都會在堆疊 (Stack) 上劃出一塊專屬區域，這就是 Stack Frame。
* 核心暫存器
- SP (Stack Pointer，堆疊指標)：永遠指向堆疊的最頂端，代表目前資料放到哪裡了。
- FP (Frame Pointer，基礎指標)：指向當前函數 Stack Frame 的「底座」。函數內的參數和區域變數，都是透過「FP 的相對位置（偏移量）」來尋找的。
運作流程解析：以 result = add(10, 20) 為例
整個函數呼叫過程可分為三個嚴格的階段：
* 階段一：主程式準備與呼叫 (Caller)
1. 壓入參數：主程式將參數依序放入堆疊。
- PUSH 10
- PUSH 20
2. 執行呼叫 (CALL)：
- CALL add_label
- 關鍵動作：CALL 指令在跳轉前，會自動將「下一行指令的位址（Return Address）」壓入堆疊。這確保了函數執行完後知道要回哪裡。
* 階段二：進入函數 (Prologue / 序言)
程式碼進入 add 函數，必須先為自己建立 Stack Frame。
1. 保存舊狀態：
- PUSH FP （把主程式的 FP 存起來，準備將來還原）
2. 建立新基準：
- MOV FP, SP （將 FP 移到目前 SP 的位置，確立新函數的底座。此時 FP-1 是返回位址，FP-2 是參數 20，FP-3 是參數 10）
3. 分配區域變數：
- SUB SP, 1 （將 SP 往上提，為區域變數 sum 預留空間）
* 階段三：執行邏輯與返回 (Epilogue / 結語)
1. 執行運算：函數透過 FP 找到 10 與 20，相加後存入預留的 sum 空間。最後將結果準備好（通常放在特定暫存器或堆疊頂部）。
2. 拆除 Stack Frame：
- MOV SP, FP （清空所有區域變數）
- POP FP （彈出剛才保存的舊 FP，恢復主程式的基準點）
3. 返回原處 (RET)：
R- ET
- 關鍵動作：RET 指令會從堆疊彈出「Return Address」，並跳回主程式繼續執行。
