## 遊戲規則

1. 蛇自動朝目前方向前進
2. 控制蛇頭吃到紅色蘋果即可得分
3. 每吃一顆蘋果，蛇身增長一節，分數 +1
4. 撞到牆壁或撞到自己身體即 Game Over
5. Game Over 後可按「重新開始」再玩一次

## 視覺設計

### 蛇
- **橢圓形立體蛇節** — 每節以橢圓繪製，非單純方塊
- **漸層配色** — 蛇頭亮綠、蛇尾漸變成黃綠( `rgb(30→120, 180→120, 30→70)` )
- **眼睛系統** — 白色眼白 + 黑色瞳孔，會依據行進方向轉向（左右移動時眼睛在頭部兩側，上下移動時在頭部上下方）
- **平滑插值移動** — 蛇節不跳格，60fps 流暢滑動

### 蘋果
- **圓形紅色果實** — 深紅色主體 + 深紅邊框
- **反光效果** — 左上角粉紅色橢圓反光，增加立體感
- **咖啡色梗** — 細線繪製
- **綠色葉子** — 橢圓形葉片

### 背景
- **深色主題** — 深藍色底色 (`#1a1a2e`)
- **網格線** — 30×30 的暗色網格，輔助視覺定位

### Game Over 面板
- 半透明深色遮罩
- 紅色邊框 + 紅色大標題
- 顯示最終得分
- 提示重新開始

## 技術架構

### 網頁版（snake.html）

#### 渲染引擎
- **Canvas 2D API** — 直接操作像素緩衝區，無 DOM 節點開銷
- **`requestAnimationFrame`** — 瀏覽器原生幀率控制，自動對齊螢幕更新率（通常 60Hz）

#### 遊戲循環
```
requestAnimationFrame(loop)
    │
    ├─ 檢查是否該移動蛇 (elapsed >= MOVE_MS)
    │   ├─ 是：呼叫 moveSnake() → 更新遊戲狀態
    │   └─ 否：跳過
    │
    └─ 呼叫 draw()
        ├─ 清除畫布
        ├─ 繪製背景 + 網格
        ├─ 繪製蘋果
        ├─ 繪製蛇（含插值 lerp）
        └─ 如果 Game Over，繪製遮罩
```

#### 平滑插值演算法
```
每幀計算 lerp = min(經過時間 / MOVE_MS, 1.0)
          (0.0 ← 剛移動完成 → 1.0 ← 即將下一次移動)

蛇節 i 的位置 = oldSnake[i] + (snake[i] - oldSnake[i]) × lerp

蛇頭眼睛位置 = snake[0] - moveDir × (1 - lerp)
```

#### 鎖定移動方向
```
moveDir 在 moveSnake() 時凍結（= 當時的 direction）
避免轉彎時蛇頭瞬移
```

#### 鍵盤輸入
- `keydown` 事件監聽
- 方向鍵即時更新 `direction`
- `e.preventDefault()` 防止頁面滾動
- 180 度迴轉保護：`d[0] !== -direction[0] || d[1] !== -direction[1]`

### Python 版（snake.py）

#### 渲染引擎
- **tkinter Canvas** — Python 標準 GUI 函式庫

#### 效能優化（共 6 輪迭代）
| 問題 | 解決方案 |
|------|----------|
| 每幀 delete + create 900+ 物件 | 改用 `coords()` + `itemconfig()` |
| 預先建立 900 個蛇節浪費記憶體 | 動態物件池，蛇多長就建多少 |
| 按鈕點擊後鍵盤失靈 | `canvas.focus_set()` 確保 focus |
| 蛇一格一格跳、延遲感 | 分離遊戲邏輯 (200ms) 與渲染 (60fps) |
| 轉彎時蛇頭瞬移 | `move_dir` 鎖定插值方向 |
| 眼睛突然出現 | 初始化就建好眼睛 |

#### 關鍵程式碼結構
```python
class SnakeGame:
    ├─ __init__()         # 初始化 UI、背景、眼睛、overlay
    ├─ build_bg()         # 繪製背景 + 網格 + 食物元件
    ├─ build_eyes()       # 預先建立 4 個眼睛 canvas 物件
    ├─ build_overlay()    # 預先建立 Game Over 面板
    ├─ restart()          # 重置遊戲狀態
    ├─ ensure_segs(n)     # 動態確保有 n 個蛇節物件
    ├─ spawn_food()       # 隨機生成食物（避開蛇身）
    ├─ change_dir(event)  # 按鍵處理（即時更新方向）
    ├─ move_snake()       # 遊戲邏輯：移動 + 碰撞 + 吃食物
    ├─ loop()             # 主循環：60fps 檢查 + 渲染
    ├─ draw_snake(lerp)   # 繪製蛇與眼睛（插值）
    ├─ draw_food()        # 繪製蘋果
    ├─ draw(lerp)         # 統整繪圖流程
    └─ game_over()        # 顯示 Game Over 面板
```

### 第 5 版 — 首次效能優化
使用者回饋：「調整偵率太卡了」

**問題分析：** 每幀 `canvas.delete("all")` + 重新繪製 900 個背景格子 + 蛇 + 食物，tkinter 不堪負荷。

**解決方案：**
```python
# 優化前（每幀執行）
canvas.delete("all")
for row in range(30):
    for col in range(30):
        canvas.create_rectangle(...)  # 900 次
draw_grid_lines()                     # 62 次
draw_snake()                         # N 次
draw_food()                          # 5 次

# 優化後（背景只畫一次，之後只更新位置）
canvas.coords(seg[i], new_x1, new_y1, new_x2, new_y2)
canvas.itemconfig(seg[i], fill=color)
```

**效果：** 背景只需建立一次（63 個 canvas 物件），之後每幀只更新蛇和食物的位置，不再 delete/create。

---

### 第 6 版 — 動態物件池 + Focus 修正
使用者回饋：「還是很卡，為甚麼」

**除錯發現的 Bug：**
1. **900 個預建蛇節** — 即使隱藏，tkinter 仍要管理這些物件
2. **按鈕 focus 竊取** — 點擊「重新開始」後，按鈕搶走 focus，方向鍵失效

**修正：**
- 改為動態蛇節池：蛇長度多少就建立多少 canvas 物件
- 每次 restart 及初始化時呼叫 `canvas.focus_set()`
- 背景從 900 個格子 + 62 條線 → 1 個底色矩形 + 62 條線

---

### 第 7 版 — 60fps 平滑插值
使用者回饋：「請把延遲處理掉」

**核心問題：** 遊戲以 200ms 間隔 tick，蛇「跳格」移動，視覺上有停頓感。

**解決方案：分離遊戲邏輯與渲染**
```
時間軸:
├───┬───┬───┬───┬───┬───┬───┬───┬───┬───┬───┬───┐
0  16  32  48  64  80  96 112 128 144 160 176 192 200 ms
│   │   │   │   │   │   │   │   │   │   │   │   │
└───渲染───┘ (60fps, every 16ms)
                    └──────遊戲邏輯──────┘ (200ms)
```

**插值公式：**
```
蛇節 i 的位置 = oldPos + (newPos - oldPos) × lerp
其中 lerp = min(經過時間 / MOVE_MS, 1.0)
```

---

### 第 8 版 — 轉彎不掉幀 + 網頁版
使用者回饋：「一轉彎就掉偵，眼睛還突然出現」

**問題分析：**
1. 方向鍵按下後 `direction` 立刻改變，但插值仍使用此方向計算蛇頭位置 → 蛇頭瞬移
2. 眼睛在 `draw_snake` 中動態建立 → 建立瞬間跳出

**修正：**
- 新增 `move_dir`：在 `move_snake()` 時凍結目前方向，插值使用 `move_dir` 而非 `direction`
- 眼睛預先建立（`build_eyes()`），永遠存在
- Game Over 時隱藏眼睛

**最終決定 — 網頁版：**
由於 tkinter 在某些環境（如 WSL X11 forwarding）無論如何優化都有延遲，決定推出 HTML/Canvas 網頁版：
- `requestAnimationFrame` → 真正 60fps
- Canvas 2D → 無 DOM 開銷
- 跨平台、免安裝、雙擊即玩
